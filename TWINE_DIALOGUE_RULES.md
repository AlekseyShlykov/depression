# Правила: из Twine в тексты для игры

Этот файл описывает, как подготовить историю из Twine, чтобы её можно было положить в `dialogues/ru.json` и `dialogues/en.json` (формат **«Twine to JSON»** с полями `cleanText`, `links`, `passageName` / `linkText`).

Разбор в коде: `js/twison.js` (парсинг текста пассажа) и `js/main.js` → `runStoryCommand` (служебные команды).

---

## 1. Имена персонажей в тексте **не пишем**

- Под портретами игрок уже видит подписи (настраиваются в `locales/<язык>.json` → `speakers.alien1`, `speakers.alien2`).
- В окне диалога показывается **только текст после двоеточия**: строка `Alien1: Смотри, он просыпается!` отображается как «Смотри, он просыпается!», без имени в колонке текста.
- **Не дублируйте** в реплике имена, клички и вводные вроде «Инопланетянин Кзых: …» или «Зорг говорит: …» — это ломает задумку интерфейса. Кто говорит, задаётся только техническим префиксом:
  - `Alien1: …` или `alien1: …` — левый персонаж;
  - `Alien2: …` или `alien2: …` — правый.
- Строка **без** префикса `Что-то:` считается **рассказчиком** (курсив, приглушённый стиль). Используйте осознанно.

---

## 2. Служебные команды: целая строка `//текст`

- Строка, которая **начинается с** `//`, а после слешей есть непустой текст — это **команда**, а не реплика.
- Она **не показывается** в диалоге и **не печатается** печатной машинкой; движок передаёт текст команды (без `//`) в обработчик.
- Пустая команда (`//` или только пробелы после `//`) — игнорируется.

Примеры:

```text
Alien1: Давайте включим музыку.
//включить музыку
```

В **английской** версии того же смысла лучше писать команды по-английски, чтобы сработали те же триггеры, что в коде:

```text
Alien1: Let's put on some music.
//play music
```

Сейчас в `runStoryCommand` распознаётся в основном **фоновая музыка** (включить / выключить), в том числе русские и английские формулировки. Точные шаблоны смотрите в `js/main.js` внутри `runStoryCommand`. Неподдержанная команда не ломает игру — пишется предупреждение в консоль браузера.

Чтобы добавить новое действие по `//…`, расширяйте `runStoryCommand` (и при необходимости документируйте здесь же).

---

## 3. Остальные соглашения по тексту пассажа

- **Одна реплика — одна строка** в Twine (один `Alien1:` / `Alien2:` / команда / строка рассказчика на строку `cleanText`).
- **Пустая строка** между строками в `cleanText` разделяет реплики: каждая непустая строка становится отдельным шагом. На экране в каждый момент показывается **только одна** реплика; после того как текст допечатан (или игрок тапнул по экрану, чтобы допечатать сразу), если дальше в этом же пассаже есть ещё реплики, появляется кнопка **«Дальше»** (`locales` → `continue`); по нажатию предыдущая строка убирается и показывается следующая.
- **Несколько абзацев в одной реплике** (одна строка Twine или один блок после `Alien1:`, внутри которого два абзаца через пустую строку `\n\n`): после разбора в `twison.js` они превращаются в **несколько последовательных** реплик того же говорящего — снова по одной на экран с кнопкой «Дальше» между ними.
- После **последней** реплики в пассаже кнопки «Дальше» нет: сразу показываются варианты **`links`** (если есть) или срабатывает конец истории.
- Ссылки на другие пассажи задаются массивом **`links`**: кнопка = `linkText`, переход = `passageName` (должно **точно** совпадать с полем `name` целевого пассажа).
- Отдельные строки вида `[[...]]` в теле не обязательны, если уже есть структурированные `links` и корректный `cleanText`.
- **Первый пассаж** в массиве `passages` — старт истории (имя может быть, например, `Start`).

---

## 4. Что сделать после экспорта из Twine

1. Вставить/обновить JSON в **`dialogues/ru.json`** (или тот язык, который вы гоните из Twine).
2. Для второго языка — **`dialogues/en.json`** с тем же графом ссылок (`passageName`), но переведёнными `cleanText`, `linkText` и командами `//…` на языке, который ожидает `runStoryCommand` (для музыки — EN-фразы из кода или зеркальные русские).
3. Проверить в браузере переключение языка и финальный пассаж без ссылок (если нужна форма email в конце).

Если структура экспорта меняется, править нужно прежде всего **`js/twison.js`** и этот файл правил.
